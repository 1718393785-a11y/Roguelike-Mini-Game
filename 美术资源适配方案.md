## 1. 已识别的设计文案内容

原方案已经覆盖 6 类核心内容：

| 模块 | 已有内容 | 可直接复用程度 |
|---|---|---|
| 武器图标 | 6 把武器 × 6 级提示词，含颜色、元素、终极形态 | 高 |
| 敌人/怪物 | 普通兵、精英、弓弩手、木牛、虎卫、木箱、5 位 Boss | 高 |
| 被动技能图标 | 10 个被动技能图标主题 | 高 |
| 页面 UI | HUD、升级面板、Boss 警告、菜单、结算 | 中 |
| 图片识别映射 | 文件命名规则 + `asset-mapping.json` 推荐方案 | 高 |
| 自动化工作流 | 生成提示词、验证资源完整性脚本思路 | 中 |

当前游戏侧已经有这些对应点：

| 资源目标 | 当前游戏位置 |
|---|---|
| 武器/升级名 | `game.js` 的 `WEAPON_UPGRADES`，以及 `src/spec/weapons.json` |
| 被动技能 | `game.js` 的 `PERK_UPGRADES`，以及 `src/spec/balance.json.passiveSkills` |
| 敌人/道具 | `Enemy` / `Boss` / `Pickup` / `DestructibleProp` 等类 |
| 渲染 | 当前实际运行以 `game.js` Canvas 绘制为主，`src/render` 是后续架构层 |
| Pixi | 已有 Pixi overlay 验证，但还没有真实贴图资产接入 |

## 2. 需要补充的关键缺口

1. 缺少玩家角色资源  
   当前方案没有主角关羽/玩家的精灵、受击、移动、攻击方向、死亡/胜利姿态。玩家是第一视觉焦点，必须补。

2. 敌人只有静态图，没有动画规范  
   怪物同屏数量多，不建议所有敌人都做复杂动画。需要拆成 `static`、`2-frame idle`、`4-frame move`、`boss animation` 四档。

3. 缺少弹体和武器运行时特效资源  
   文档有武器图标，但游戏中还需要箭矢、枪芒、刀光、剑阵、盾波、龙卷风、雷柱、火墙等战斗内资源。

4. 缺少大地图背景资源规范  
   P3 已有 6000×6000 大地图和滚动背景，后续需要地面 tile、边界标识、关卡色调 overlay、战场装饰。

5. 缺少图集/纹理烘焙策略  
   Pixi 不能每帧 new Graphics。美术接入时必须用 atlas 或初始化烘焙纹理，运行时只复用 sprite。

6. 缺少加载失败兜底标准  
   图片资源可能缺失、命名错误、尺寸不符。必须保留现有 Canvas 绘制作为 fallback。

7. 缺少资源质量验收指标  
   需要自动检查透明通道、尺寸、命名、manifest 完整性、最大文件体积、非空像素比例。

8. 缺少美术 Feature Flag 分层  
   不能只用一个总开关，否则排查困难。应该拆成总开关 + 分类开关。

## 4. Feature Flag 设计

默认全部关闭：

```javascript
const FEATURE_FLAGS = {
  ENABLE_ART_ASSETS: false,
  ENABLE_ART_WEAPON_ICONS: false,
  ENABLE_ART_ENEMY_SPRITES: false,
  ENABLE_ART_PLAYER_SPRITE: false,
  ENABLE_ART_PICKUPS: false,
  ENABLE_ART_EFFECTS: false,
  ENABLE_ART_UI_SKIN: false,
  ENABLE_ART_PIXI_TEXTURES: false,
};
```

回滚规则：

| 开关 | 影响范围 | 关闭后行为 |
|---|---|---|
| `ENABLE_ART_ASSETS` | 总开关 | 完全走现有 Canvas 绘制 |
| `ENABLE_ART_WEAPON_ICONS` | 升级面板、冷却 HUD | 回退文字/形状图标 |
| `ENABLE_ART_ENEMY_SPRITES` | 敌人绘制 | 回退矩形/圆形绘制 |
| `ENABLE_ART_PLAYER_SPRITE` | 玩家绘制 | 回退原玩家绘制 |
| `ENABLE_ART_PICKUPS` | 掉落物 | 回退发光圆点/方块 |
| `ENABLE_ART_EFFECTS` | 战斗特效 | 回退原 Canvas 特效 |
| `ENABLE_ART_UI_SKIN` | 菜单、HUD、结算 | 回退原 UI |
| `ENABLE_ART_PIXI_TEXTURES` | Pixi 真实贴图 | 回退 Pixi 白色纹理/Canvas |

## 7. 美术生成补充规范

### 7.1 统一风格基准

风格关键词：

```text
暗黑三国，俯视45度，战场可读性优先，轮廓高对比，低饱和底色 + 高亮技能色，
透明背景，游戏内小尺寸清晰，非海报，非概念图，非复杂背景。
```

统一负向提示：

```text
文字, 水印, 背景场景, 复杂构图, 低清晰度, 过度写实, 3D渲染感, 侧视, 正面肖像,
表情特写, 多角色混杂, 裁切身体, 模糊边缘, 光污染遮挡主体。
```

### 7.2 玩家资源补充

最低可用：

| 资源 | 数量 | 用途 |
|---|---:|---|
| idle | 1-2 帧 | 默认站立 |
| move | 4 帧 | 移动循环 |
| hit | 2 帧 | 受击无敌闪烁时可选 |
| dead | 1 帧 | 死亡结算 |

提示词方向：

```text
2D游戏主角精灵，关羽千里走单骑，青绿色战袍，赤兔马元素可弱化为披风/马纹，
手持青龙偃月刀但不遮挡身体，俯视45度，透明背景，暗黑三国风，高对比轮廓，
适合 64px 游戏内缩放，256x256 源图。
```

### 7.3 敌人动画分档

| 类型 | 推荐帧数 | 原因 |
|---|---:|---|
| 普通 soldier | idle 1 + move 2 | 同屏多，控制成本 |
| elite / tiger_guard | idle 1 + move 4 | 辨识度更高 |
| archer | idle 1 + shoot 2 | 强化远程行为 |
| wooden_ox | idle 1 + move 2 | 快速逃跑读得出来即可 |
| boss | idle 2 + attack 4 | Boss 数量少，可以更丰富 |
| destructible_prop | intact/cracked/broken 各 1 | 破坏反馈 |

### 7.4 特效资源补充

| 效果 | 建议资源 |
|---|---|
| 刀光 | 半透明弧形 slash sprite，可旋转缩放 |
| 枪芒 | 直线光束/枪影 sprite |
| 弩箭 | 箭矢 sprite + 雷电命中特效 |
| 飞剑 | 单把飞剑 sprite，运行时旋转 |
| 盾波 | 环形 pulse sprite 或继续 Canvas 绘制 |
| 火龙卷 | 4-6 帧旋转 strip |
| 雷柱 | 竖向雷电 strip 或九宫格拉伸 |
| 火墙 | 环形火焰 tile/segment，不做全屏大图 |

原则：运行时可旋转/缩放的资源优先做小图复用，不要每个半径都生成一张图。

## 9. 自动化验收标准

### 9.1 资源完整性

每次接入前运行：

```text
node tools\validate-assets.mjs
```

检查项：

| 检查 | 标准 |
|---|---|
| 文件存在 | manifest 中路径必须存在 |
| 尺寸 | 图标 512/256，实体源图 256，pickup 64/128 |
| 透明通道 | PNG 必须有 alpha |
| 非空像素 | 不能是空白图 |
| 文件大小 | 单图建议小于 1MB，UI 大图小于 3MB |
| 命名 | 必须匹配 manifest 与命名规范 |
| manifest | Zod 校验通过 |

### 9.2 游戏状态不变

美术开关不应改变快照：

```text
node tools\run-baseline.mjs --browser chrome --seed 12345 --record records\smoke.json --seconds 120
node tools\run-baseline.mjs --browser chrome --seed 12345 --record records\smoke.json --seconds 120 --art-assets
```

要求：

```text
玩家位置/血量/经验、敌人数量/总血量、子弹数量、击杀数、DPS 100% 一致。
```

### 9.3 性能

最低标准：

```text
npm run validate:entity-stress
```

要求：

```text
2000 实体压力测试通过；启用图片资源后不能显著低于 Canvas fallback。
```

### 9.4 视觉 QA

Chrome 截图检查：

| 场景 | 要求 |
|---|---|
| 开始菜单 | 标题与按钮无遮挡 |
| 普通战斗 | 玩家、敌人、掉落物可读 |
| 大地图 | 视野外资源不绘制 |
| 升级面板 | 图标与文字不重叠 |
| Boss 入场 | Boss 与警告 UI 可读 |
| 结算 | 统计文字清晰 |

## 10. 推荐优先级

最稳妥顺序：

1. `assets/style-bible.md` + manifest 空壳。
2. prompts 自动生成工具。
3. validate-assets 校验工具。
4. AssetLoader 加载不渲染。
5. 武器/被动图标。
6. 拾取物/木箱。
7. 敌人静态精灵。
8. 玩家精灵。
9. 战斗特效。
10. UI 皮肤。
11. 大地图 tile。
12. Pixi 贴图后端。

不建议先做：

| 内容 | 原因 |
|---|---|
| 一次性替换所有实体 | 问题难定位，回滚粒度太粗 |
| 先做复杂动画 | 资源一致性和性能风险高 |
| 先做 Pixi 全贴图 | 当前主运行仍是 `game.js` Canvas，容易返工 |
| 用整张 6000×6000 背景 | 内存不可控，必须 tile 化 |
| 让 AI 只按文件名自动猜资源 | manifest 更稳定，可校验 |
